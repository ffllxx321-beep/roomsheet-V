# RoomManager 编译指南

> 窝窝整理，确保一次编译成功 🎀

---

## ⚠️ 重要：Revit API 引用问题

Revit API 不是 NuGet 包，是本地 DLL 引用。Visual Studio 打开项目后，需要手动确认引用是否正确加载。

---

## 📋 编译步骤（必须按顺序执行）

### 步骤 1：打开项目
```
1. 解压 RoomManager_v0.1.0_20260422.zip
2. 用 Visual Studio 2022 打开 RoomManager.csproj
   （注意：是 .csproj 文件，不是 .sln 文件）
```

### 步骤 2：检查 Revit API 引用 ⚠️ 关键步骤
```
1. 在解决方案资源管理器中，展开"依赖项" → "程序集"
2. 检查以下引用是否有黄色警告图标：
   - RevitAPI
   - RevitAPIUI
   - AdWindows
   - UIFramework

3. 如果有黄色警告，说明引用未加载，执行步骤 3
```

### 步骤 3：重新添加 Revit API 引用
```
1. 右键"依赖项" → "添加项目引用"
2. 点击"浏览"按钮
3. 导航到 C:\Program Files\Autodesk\Revit 2026\
4. 选择以下 DLL（按住 Ctrl 多选）：
   - RevitAPI.dll
   - RevitAPIUI.dll
   - AdWindows.dll
   - UIFramework.dll
5. 点击"添加"
6. 点击"确定"

或者直接删除旧引用，重新添加：
1. 右键旧的 RevitAPI 引用 → "移除"
2. 重复上述步骤 3
```

### 步骤 4：还原 NuGet 包
```
1. 右键解决方案 → "还原 NuGet 包"
2. 等待还原完成（输出窗口会显示进度）
```

### 步骤 5：设置编译配置
```
1. 菜单：生成 → 配置管理器
2. 设置：
   - 活动解决方案配置: Release
   - 活动解决方案平台: x64
   - 项目平台: x64
3. 点击"关闭"
```

### 步骤 6：编译项目
```
1. 菜单：生成 → 重新生成解决方案
   或快捷键：Ctrl+Shift+B
2. 检查输出窗口，确认编译成功
```

---

## 🔍 验证引用是否正确

在 Visual Studio 的"对象浏览器"中：
```
1. 菜单：视图 → 对象浏览器
2. 展开程序集列表
3. 找到 RevitAPI
4. 展开命名空间 Autodesk.Revit.DB
5. 搜索 "Room" 类
6. 如果能找到，说明引用正确
```

---

## ❌ 常见错误排查

### 错误 1：未能找到类型或命名空间名"Room"
**原因**：Revit API 引用未正确加载

**解决**：
1. 检查引用是否有黄色警告图标
2. 重新添加 Revit API 引用（步骤 3）
3. 重启 Visual Studio

### 错误 2：类型冲突（View、ListBox 等）
**原因**：项目配置了 UseWindowsForms

**解决**：已修复，确保使用最新代码

### 错误 3：缺少 using 引用
**原因**：文件头部缺少命名空间引用

**解决**：已修复，确保使用最新代码
