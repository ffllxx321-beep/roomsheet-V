## 1. 项目配置规范

### ✅ 正确配置
```xml
<PropertyGroup>
  <TargetFramework>net8.0-windows</TargetFramework>
  <UseWPF>true</UseWPF>
  <!-- ❌ 不要加这行！会导致类型冲突 -->
  <!-- <UseWindowsForms>true</UseWindowsForms> -->
</PropertyGroup>
```

### ⚠️ 为什么不能用 UseWindowsForms？

| 冲突类型 | WPF | WinForms |
|---------|-----|----------|
| `View` | `Autodesk.Revit.DB.View` | `System.Windows.Forms.View` |
| `ListBox` | `System.Windows.Controls.ListBox` | `System.Windows.Forms.ListBox` |
| `KeyEventArgs` | `System.Windows.Input.KeyEventArgs` | `System.Windows.Forms.KeyEventArgs` |

**结论**：WPF 项目不要启用 WinForms，除非真的需要用 WinForms 控件。

## 2. Using 引用规范

### 必须检查的引用

| 使用的类型 | 必须添加的 using |
|-----------|-----------------|
| `Room` | `using Autodesk.Revit.DB.Architecture;` ⚠️ |
| `RoomData`, `FieldDefinition` | `using RoomManager.Models;` |
| `RelayCommand` | `using RoomManager.Utils;` |
| 各种 Service 类 | `using RoomManager.Services;` |
| `Room`, `Document`, `View`, `Element` | `using Autodesk.Revit.DB;` |
| `ObservableCollection` | `using System.Collections.ObjectModel;` |
| `INotifyPropertyChanged` | `using System.ComponentModel;` |
| `ICommand` | `using System.Windows.Input;` |
| `MessageBox` | `using System.Windows;` |
| `File`, `Directory`, `Path` | `using System.IO;` |
| LINQ 方法 | `using System.Linq;` |

### 自动检查脚本

```bash
# 检查缺失的 using 引用
for f in $(find . -name "*.cs" -not -path "./obj/*"); do
  if grep -qE '\bRoomData\b' "$f" && [[ "$f" != *"/Models/"* ]]; then
    grep -q "using RoomManager.Models" "$f" || echo "❌ $f 缺少 using RoomManager.Models"
  fi
done
```

## 5. 常见错误复盘

### 错误 1：类型冲突
**症状**：`"View"是"Autodesk.Revit.DB.View"和"System.Windows.Forms.View"之间的不明确的引用`

**原因**：项目配置了 `<UseWindowsForms>true</UseWindowsForms>`

**解决**：移除 `UseWindowsForms` 配置

---

### 错误 2：缺失 using 引用
**症状**：`未能找到类型或命名空间名"RoomData"`

**原因**：文件头部缺少 `using RoomManager.Models;`

**解决**：添加对应的 using 引用

---

### 错误 3：重复代码片段
**症状**：`成员声明中的标记"("无效`

**原因**：代码复制粘贴时产生重复片段

**解决**：仔细检查代码结构，删除重复部分

---

### 错误 4：Revit API 类型找不到
**症状**：`未能找到类型或命名空间名"Room"`

**原因**：Revit API DLL 引用路径不正确

**解决**：
1. 确认 Revit 已安装在 `C:\Program Files\Autodesk\Revit 2026\`
2. 运行 `检查Revit安装.ps1` 脚本检测安装路径
3. 修改 `.csproj` 中的 `<HintPath>` 为正确路径

---

### 错误 5：嵌套类型访问错误
**症状**：`类型"DwgRecognitionService"中不存在类型名"MatchStatus"`

**原因**：`MatchStatus` 是独立枚举，不是 `DwgRecognitionService` 的嵌套类型

**解决**：使用 `MatchStatus` 而不是 `DwgRecognitionService.MatchStatus`

---

### 错误 6：Room 类型找不到 ⚠️ 最常见
**症状**：`未能找到类型或命名空间名"Room"`

**原因**：`Room` 类在 `Autodesk.Revit.DB.Architecture` 命名空间，不在 `Autodesk.Revit.DB`

**解决**：添加 `using Autodesk.Revit.DB.Architecture;`

```csharp
// ❌ 错误
using Autodesk.Revit.DB;

// ✅ 正确
using Autodesk.Revit.DB;
using Autodesk.Revit.DB.Architecture;
```
