## 🔴 问题 1：Revit 2026 API 返回类型变化

### 错误信息
```
CS0030: 无法将类型"Autodesk.Revit.DB.ElementId"转换为"Autodesk.Revit.DB.Element"
```

### 错误位置
`RevitRoomService.cs` 第 120 行

### 根本原因
**Revit 2026 API 变更**：`Document.Create.NewRooms2(Level, Phase)` 返回类型从 `ElementSet` 改为 `ISet<ElementId>`

### 错误代码
```csharp
// ❌ 错误：Revit 2026 中 NewRooms2 返回 ElementId 集合
var roomElements = _document.Create.NewRooms2(level, phase);
foreach (Element element in roomElements)
{
    if (element is Room room)
```

### 正确代码
```csharp
// ✅ 正确：先获取 ElementId，再通过 GetElement 获取 Element
var roomElementIds = _document.Create.NewRooms2(level, phase);
foreach (ElementId elementId in roomElementIds)
{
    if (_document.GetElement(elementId) is Room room)
```

### 预防措施
1. **升级 Revit 版本前，必须查阅 API 变更文档**
   - Revit 2026 API Changes: https://www.revitapidocs.com/
   - 搜索 "Breaking Changes" 或 "API Changes"

2. **遇到类型转换错误时，检查返回类型**
   - 使用 `var` 时要特别注意，IDE 可能无法推断正确类型
   - 明确指定类型：`ISet<ElementId> roomElementIds = ...`

3. **使用官方 API 文档验证**
   - https://www.revitapidocs.com/
   - 搜索方法名，查看不同版本的签名

## 🔴 问题 2：NuGet 包版本号错误

### 错误现象
编译成功，但 `bin` 文件夹中缺少依赖 DLL（只有 `RoomManager.dll` 和 `RoomManager.pdb`）

### 根本原因
**NuGet 包版本号写错**，Visual Studio 无法下载正确的包

### 错误配置
```xml
<PackageReference Include="ACadSharp" Version="1.0.0" />
<PackageReference Include="EPPlus" Version="7.0.0" />
<PackageReference Include="itext7" Version="8.0.0" />
```

### 正确配置
```xml
<PackageReference Include="ACadSharp" Version="3.4.29" />
<PackageReference Include="EPPlus" Version="8.5.3" />
<PackageReference Include="itext7" Version="9.6.0" />
```

### 预防措施
1. **添加 NuGet 包前，先在 NuGet Gallery 查询最新版本**
   - https://www.nuget.org/packages/ACadSharp
   - https://www.nuget.org/packages/EPPlus
   - https://www.nuget.org/packages/itext7

2. **使用 Visual Studio 的 NuGet 包管理器**
   - 右键项目 → 管理 NuGet 程序包
   - 搜索包名，选择最新稳定版本
   - 不要手动写版本号

3. **编译后检查 bin 文件夹**
   - 应该包含所有依赖 DLL
   - 如果缺少，检查 NuGet 包是否正确安装

## 🔴 问题 3：依赖 DLL 未复制到输出目录

### 错误现象
编译成功，但 `bin` 文件夹中缺少 NuGet 包的 DLL

### 根本原因
**.NET SDK 风格项目默认不复制 NuGet 依赖到输出目录**

### 解决方案
在 `.csproj` 中添加：
```xml
<PropertyGroup>
  <CopyLocalLockFileAssemblies>true</CopyLocalLockFileAssemblies>
</PropertyGroup>
```

### 预防措施
1. **Revit 插件项目必须添加此配置**
   - Revit 插件需要将所有依赖 DLL 复制到 Revit Addins 目录
   - 没有此配置，只有主 DLL 会被复制

2. **编译后检查 bin 文件夹内容**
   - 应该包含：主 DLL + 所有 NuGet 依赖 DLL
   - 如果缺少，添加 `CopyLocalLockFileAssemblies`

## 🔴 问题 4：缺少 System.Drawing.Common 引用

### 错误信息
```
CS0246: 找不到类型或命名空间名"Color"(是否缺少 using 指令或程序集引用?)
```

### 根本原因
**.NET 8 中 `System.Drawing` 需要单独的 NuGet 包**

### 解决方案
```xml
<PackageReference Include="System.Drawing.Common" Version="8.0.0" />
```

### 预防措施
1. **.NET Core / .NET 5+ 项目中，System.Drawing 不再内置**
   - 需要手动添加 `System.Drawing.Common` NuGet 包

2. **使用 EPPlus 等依赖 System.Drawing 的库时，必须添加此包**

## 🔴 问题 5：iText 9.x API 变化

### 错误信息
```
CS1061: "Paragraph"未包含"SetBold"的定义
```

### 根本原因
**iText 9.x API 变更**：`SetBold()` 方法签名改变

### 错误代码
```csharp
// ❌ iText 7.x 用法
new Paragraph("标题").SetBold()
```

### 正确代码
```csharp
// ✅ iText 9.x 用法
new Paragraph("标题").SetBold(true)
```

### 预防措施
1. **升级 NuGet 包版本前，必须查阅迁移指南**
   - iText 7 → 8: https://itextpdf.com/blog/itext-8-migration-guide
   - iText 8 → 9: https://itextpdf.com/blog/itext-9-migration-guide

2. **搜索 "breaking changes" 或 "migration guide"**
   - Google: "itext 9 breaking changes"
   - 官方文档：Release Notes

3. **使用保守的版本号**
   - 如果不需要新功能，不要升级到最新版本
   - 使用已知稳定的版本（如 itext7 8.0.5）

4. **升级后全面测试**
   - 编译通过不代表 API 兼容
   - 运行时可能抛出异常

## 📝 总结

### 核心教训

1. **升级依赖前必须查阅迁移指南**
   - 不要盲目升级到最新版本
   - Breaking Changes 可能导致编译错误或运行时错误

2. **使用官方文档验证 API**
   - Revit API: https://www.revitapidocs.com/
   - NuGet 包: https://www.nuget.org/

3. **Revit 插件项目需要特殊配置**
   - `CopyLocalLockFileAssemblies=true`
   - Revit DLL 引用 `Private=false`

4. **编译后必须检查输出目录**
   - 确保所有依赖 DLL 都被复制
   - 缺少 DLL 会导致 Revit 加载失败
